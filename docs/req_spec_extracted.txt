提交後 GAS 自動執行： 

•  於 Classes Sheet 建立班級記錄 
•  依開課日至結課日，計算所有符合星期設定的上課日，寫入 Sessions Sheet 
•  批次建立 Google Calendar 事件（含班級名稱、教練、教室、學員名單於描述欄） 
•  將 Calendar 事件 ID 回寫至 Sessions / Classes Sheet 
•  批次建立 Enrollments 記錄 
•  於教練共享日曆建立對應事件 

3.1.2 課程調整功能 

支援以下調整類型： 

•  單堂取消（如：教練請假）：更新指定 Session status 為 cancelled，Google 

Calendar 事件標記取消，觸發公告推播 

•  單堂改期：更新指定 Session 日期，更新 Calendar 事件時間 
•  全班停課（如：颱風）：批次更新該班所有未來 Sessions，觸發全班公告 
•  所有班級停課：批次更新全部班級當日 Sessions，觸發全體公告

每次調整完成後自動觸發公告推播（詳見 3.1.3）。 

3.1.3 公告推播功能 

使用 LINE Bot Reply API 搭配以下兩種觸發方式： 

1.  管理後台手動發送：管理員於後台填寫公告內容，選擇推播範圍（全體 / 指定班級 

/ 指定學員），點擊發送 

2.  課程調整自動觸發：課程調整儲存後自動發送受影響學員及教練的通知 

技術實作說明： 

LINE Reply API 需搭配 Webhook 觸發，對於非互動觸發的主動推播，改採 LINE 
Messaging API 的 multicast 或 broadcast 功能（免費方案每月 200 則免費，超出以每則 
0.x 元計，需評估用量）。若免費額度不足，建議以「管理員在 LINE Bot 中操作選單 → 
Bot 回傳公告 → 管理員確認 → Reply API 回覆」作為替代方案。 

3.2 教練端（共享 Google Calendar） 

GAS 開班後自動將教練共享日曆事件建立完成。教練可透過 Google Calendar App（手機 
/ 電腦）查看。 

每個日曆事件描述欄自動帶入： 

•  班級名稱、日期、教室 
•  應到學員名單（姓名清單） 
•  請假學員名單 
•  補課學員名單（含來源班級） 

教練異常回報功能（預留，視難易度實作）： 

•  日曆事件描述欄預留「回報連結」欄位，連結至 GitHub Pages 的回報表單 
•  教練填寫實際出席與日曆不符的情況（如：學員實際出席但系統顯示請假） 
•  管理員審核後，由 GAS 更新 Attendance Sheet 對應記錄 

3.3 LINE Bot 學員端 

3.3.1 身份綁定 

學員加入 LINE 官方帳號時，Bot 自動發送歡迎訊息與綁定說明： 

•  學員點選「綁定身份」按鈕，Bot 引導輸入姓名 + 手機後四碼（或唯一驗證碼） 
•  GAS 比對 Students Sheet，找到對應學員後將 LINE UID 寫入 
•  完成綁定後發送確認訊息及個人資訊卡

•  補課跨班：系統標記學員出席於補課班的對應 Session 
•  補課有效期限：當期結束前（可於 SystemConfig 調整）

4. 功能規格 — 進階版 

4.1 線上報名系統 

•  管理後台提供公開報名頁面連結（GitHub Pages） 
•  學員填寫個人資料 + 選擇課程 → GAS 建立 Enrollment 記錄（status = pending） 
•  管理員審核 → 確認後狀態改為 active，觸發通知學員 
•  報名人數超過上限時，自動顯示「候補」並通知管理員 

4.2 教練鐘點計算 

•  每堂課結束後，GAS 依 Sessions + Coaches 資料計算應付鐘點 
•  管理後台提供月結報表（依教練、依班級彙總） 
•  支援代課費率調整 

4.3 學員收費系統 

•  學費設定於 Classes Sheet（課程費率欄位） 
•  每期開課後自動建立應繳記錄，管理員確認收款後更新狀態 
•  管理後台提供應收 / 已收帳款報表 

4.4 日曆雙向同步 

•  教練修改 Google Calendar 事件描述後，透過 GAS Trigger 偵測變更 
•  GAS 解析變更內容，更新對應 Sessions / Attendance 資料 
•  需使用 Google Calendar API Push Notifications 或定時輪詢實作
