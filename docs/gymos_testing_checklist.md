# GymOS v3.0 全功能聯調測試清單

本清單列出 GymOS v3.0 系統所有需要聯調測試的核心與細節功能。測試順序嚴格依照業務邏輯的先後關係（必須先開班，才能請假與點名）。

## 🟡 階段一：管理員核心前置作業 (最優先測試)
*說明：沒有課堂，就無法產生後續所有的業務操作。因此必須先由管理員進行排程。*

- [ ] **權限升級測試**：將自己的 `line_uid` 寫入 `Staff` 表並設定為 `admin`，點選圖文選單觸發自動切換為「管理員專屬選單」。
- [ ] **進入管理後台**：透過管理員選單點擊進入 LIFF Web App (切換至 Admin Mode)。
- [ ] **新增班級排程 (Class & Session Engine)**：
  - [ ] 在網頁上輸入「班級名稱」(如: 壺鈴基礎班)。
  - [ ] 選擇「難度等級」(如: L1) 及「班級類型」。
  - [ ] 設定上課時間、開班首日與「課程總週數」(如: 12週)。
  - [ ] 點擊送出，驗證系統是否正確在 `Classes` 表建立一筆資料。
  - [ ] **核心驗證**：檢查 `Sessions` 表是否瞬間展開相對應數量的課堂資料（日期需自動計算遞增）。
- [ ] **預先登記學員**：在後台隨便輸入一位虛擬學員的姓名與生日，送出登記。

---

## 🟡 階段二：學員端核心業務 (Student Portal)
*說明：以剛才開好的課堂與時段，進行學員端的各項操作。*

- [x] **學員註冊綁定**：(已於今日完成) LIFF 安全連線、大小寫問題修復、真實 UID 完美寫入。
- [ ] **進入線上請假模組**：點擊選單進入請假頁面。
  - [ ] 介面驗證：確認畫面上正確抓出該學員「未來 4 週內」的課堂列表。
  - [ ] 操作驗證：點選其中一堂課，輸入請假原因並送出。
  - [ ] 資料驗證：確認 `Leaves` 表新增一筆狀態為 `pending` (待補課) 的請假紀錄。
- [ ] **進入跨班補課模組**：點擊選單進入補課頁面。
  - [ ] 第一步驗證：畫面正確列出剛才那筆未使用的「請假額度」。
  - [ ] 第二步動態過濾：選擇額度後，系統必須只顯示「難度相同 (例如同為 L1)」、「未來日期」且「有剩餘名額」的課堂。
  - [ ] 確認預約：點擊送出後，系統將該學員以 `makeup` 身分寫入該課堂名單。

---

## 🟡 階段三：教練端核心業務 (Coach Portal)
*說明：教練出勤點名「一律在 LINE 的 LIFF 網頁中集中操作」，不需開啟日曆。這是為了讓教練在現場能用手機最快速度點名。*

- [ ] **教練權限升級**：將另一個測試帳號寫入 `Staff` 並設為 `coach`，驗證圖文選單是否切換。
- [ ] **進入教練出勤校正中心**：透過圖文選單進入教練端網頁。
  - [ ] 課堂讀取：驗證下拉式選單是否有讀出該教練負責的「今日/近期課堂」。
  - [ ] 名單讀取：選擇課堂後，下方正確列出該堂課的所有正式生與跨班補課生，並清楚標示身分。
  - [ ] **現場點名校正**：
    - [ ] 將全勤名單中的某一位學生標記為「曠課」。
    - [ ] 點擊送出，驗證系統是否正確扣除該名學員的總額度或懲罰機制。
    - [ ] 驗證該堂課的 Session 狀態被標記為 `completed` (已結算)。
