# GymOS 正式交付檢查清單

本清單用於交付健身房客戶、正式部署或重要版本更新前的最後確認。

## 1. 範圍確認

- 確認客戶名稱、品牌名稱、Logo 與 LINE 官方帳號正確。
- 確認本次更新屬於 UI 調整、流程調整、資料結構調整或 LINE 訊息調整。
- 確認本機輔助檔、scratch 測試檔不會被加入 Git。

## 2. 本機品質檢查

- 執行 `npm.cmd run tc`。
- 開啟本機 Web App，確認學員端可正常顯示。
- 切換管理端，確認首頁摘要、課程列表、班級經營、繳費確認、課表檢視都能正常載入。
- 檢查手機寬度下的 LIFF 畫面是否擠壓或文字重疊。
- 確認沒有因 LIFF、Alpine 或 API 初始化造成空白頁。

## 3. 學員流程檢查

- 學員綁定：填寫基本資料、選擇課程類型、選擇班級時段並送出。
- 我的課表：確認剩餘堂數、已出席、請假、補課、可補額度顯示正確。
- 請假申請：確認可請假課程列表正常，送出後提示清楚。
- 補課預約：確認可用補課額度與可補課時段顯示正確。
- 學員加選：確認可加選、額滿、等級提醒與待繳費狀態。

## 4. 管理端流程檢查

- 首頁摘要：確認公告、四格數據卡與展開明細。
- 課程列表：確認已開課程、狀態標籤、續期與結束操作。
- 班級經營：確認臨時調課與新增班級表單。
- 繳費確認：確認待繳費名單與批量確認操作。
- 課表檢視：確認日曆載入與課程狀態顏色可讀。

## 5. LINE 檢查

- 確認 LIFF URL 可在 LINE 內開啟，也可在外部瀏覽器測試。
- 確認 LINE 圖文選單連到正確模式。
- 確認學員訊息文案清楚、按鈕正確。
- 確認管理端推播不暴露不必要的個資。
- 啟用大量推播前，確認 LINE 推播額度與管理端推播開關。

## 6. 資料與權限檢查

- 確認 GAS Script Property 的 `SPREADSHEET_ID` 指向正式客戶表單。
- 確認 Config 表中的 LINE Channel Access Token 與 LIFF ID 是正式資料。
- 確認 Google Sheets 與 Calendar 擁有者、共用權限符合交付安排。
- 確認 Apps Script Web App 的存取設定與後端權限檢查相互搭配。
- 確認正式環境不使用測試身分或假學員資料。

## 7. 部署

- 執行 `npm.cmd run deploy`。
- 確認 `clasp push` 成功。
- 確認 `clasp deploy -i ...` 是在既有部署下新增版本，不是新增部署。
- 確認 Web App URL 維持不變。
- 確認 Git commit 與 push 成功。

## 8. 部署後冒煙測試

- 開啟正式 Web App URL。
- 從 LINE 內開啟 LIFF URL。
- 確認管理端與學員端都能載入。
- 執行一個不破壞資料的查詢或檢視流程。
- 記錄 GAS 版本號與 Git commit hash。

## 9. 客戶交付

- 提供客戶實際需要使用的 Web App / LIFF 入口。
- 提供最新管理員操作重點。
- 列出已知限制或本次刻意延後的事項。
- 確認上線後支援窗口與聯絡方式。
