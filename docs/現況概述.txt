1_學員在100人左右,教練7人,管理2人,目前使用的管理工具（紙本、LINE群、Excel…）
2_健身訓練以徒手訓練為主,搭配啞鈴,壺鈴,桿鈴(架),單桿,深蹲架,彈力帶,彈力球,TRX,6角桿,....等輔助器材,

學員程度:從L1~10分為10階,課程難度:區分初級,中級,高級三階

班級時數:固定時段:
A:1週1堂,1堂1小時,一期12週(12堂),
B:1週2堂,1堂1小時,一期1個月(24堂)...等,不限制學員報名幾種課程 

#預劃包含以下功能:
基礎版:
1_管理端(Web App ,使用GitHub Pages建置管理員權限管理靜態網站頁面,綁定LINE或GOOGLE帳號登入,手機電腦都可以操作),(搭配 Tailwind CSS + Alpine.js + Flatpickr ,JEKYLL/hugo 快速套件健身房主題?Font Awesome,Animate.css,DataTables jQuery,LockService ,CacheService):

1.有開班設定功能(新增課程):設定班級名稱,開班起訖日期,課程起訖時間,選定使用教室,選定人數上限,套用上期學員名單,新增學員名單...填選完成提交後,試算表能對應課程的週期自動依據需求帶入相關資料及週數,完成頁面建置,做為以後該班報名,請假,補課,教練及學員查詢及日曆顯示的基礎...
2.有課程調整功能(設定停課):如遇特殊情況,如教練請假,颱風停課...等等臨時狀況,能設定調整單一班級或所有班級的課程周期...並連動公告功能(不使用line notify(因已停用),不使用LINE PUSH(因免費額度僅200則不夠使用,使用Reply API（被動觸發） 或類似的免費功能)(LIFF 置頂公告 )
3.會員管理: 可檢視日曆,課程列表,學員資訊,請假及補課狀況...等完整的後台功能(統計報表 • 請假,補課管理 )
4.有可連動使用LINE Bot Reply API 發免費的群公告及個人公告功能,(也可以在LINE BOT中直接操作)
4.保留未來擴充功能:計算教練鐘點,學員學費,報名系統.... 

2_教練端:使用共享日曆,可自動帶入試算表相關資料,內容包含該堂課學員名單,請假名單,補課名單,...等完整資訊,教練在手機上開啟日曆，就能看到今天有幾個人、誰請假、誰是來補課的。學員及教練均不用點名或打卡打卡報到,日曆中有校正或回報功能,教練針對異常或日曆資訊與上課實況不符時的回報與校正,校正後可同步更新試算表內原資料,

3_學員端(在LINE  Bot 及 web app 均可操作):
1.學員新加入健身房官方LINE時同時綁定功能(Firebase Auth ),並請學員輸入姓名,生日,
2.製作選單,選單卡片中有學員請假,補課功能及學員個人資訊卡功能(顯示該學員班級課程資訊,課程時數,剩餘堂數 ,請假時數,補課...等相關完整資訊,以 Flex Message 呈現 ) 
3.請假及補課原則:(請假補課流程設計成 Bubble Message 步驟確認，減少學員操作錯誤 )
請假:該堂課下課前均可請假,請假無上限,請假課程均可補課,補課時限在該期最後一堂課前,該期未上完課程可折抵下期學費(缺課1~２堂下期全額折抵,缺課第3堂以後以原價75折計算折抵),全勤者下期95折
補課:可跨班,但補課要區分程度.原則上僅能參加同程度的課程補課,補課登記後不可再更改,不可取消,補課缺課不得再補,

4_其他:
1.錯誤記錄與監控：在 GAS 中建立一個 ErrorLog Sheet，所有 Webhook 錯誤、API 呼叫失敗、Trigger 異常都自動寫入，管理員可直接在 Sheet 看到問題，不需要進入 GAS 控制台。
2.所有重要ID都寫入GAS專案屬性中,並建立一張 SystemConfig Sheet，存放健身房名稱、LINE Channel Access Token、日曆 ID 等設定，避免硬編碼。未來部署新客戶時，只需複製整份 Spreadsheet 並更新設定值，不需要修改程式碼。
